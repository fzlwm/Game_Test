# 游戏测试作品集
(注意:Asset下有bug复现视频,工具操作记录,部分测试用例图)

## 项目概述
本项目展示了我在游戏测试领域的专业技能和经验，包含了多个游戏项目的测试案例、缺陷报告和性能测试分析。

## 测试理念
- **用户体验至上**：从玩家角度出发，关注游戏的整体体验
- **全面覆盖**：确保测试覆盖游戏的各个功能模块
- **严谨细致**：注重细节，发现潜在的问题
- **持续改进**：通过测试反馈推动游戏质量提升

## 技能标签
- 黑盒测试
- 功能测试
- 性能测试
- 弱网环境测试
- API测试
- 安全测试
- 缺陷分析与报告
- 测试用例设计
- 探索性测试

## 项目结构

### 1. 游戏缺陷报告
- `01_MyGame_Bug/`：包含游戏中的Bug完整报告，通过详细的测试步骤和截图记录问题

### 2. 抽卡模块测试
- `02_StarRail_Draw_Module/`：针对抽卡系统的功能测试，设计了30个测试用例，覆盖了各种场景和边界情况
- **测试过程**：从功能测试入手，逐步扩展到异常场景和边界条件，确保抽卡系统的稳定性和用户体验
- **考量**：重点关注抽卡概率、资源消耗、背包管理等核心功能

### 3. MoBa游戏英雄测试
- `03_Hero_Lixin_Modele/`：对李信英雄的技能、建模、语音等方面进行全面测试
- **测试过程**：从英雄建模入手，逐步测试技能特性、特效表现、语音触发等多个维度
- **考量**：注重英雄的平衡性和玩家操作体验

### 4. 弱网环境测试
- `Weak_Network_Test/`：使用Clumsy工具模拟弱网环境，测试游戏在不同网络条件下的表现
- **测试过程**：通过模拟延迟、丢包等网络问题，验证游戏的稳定性和容错能力
- **考量**：确保玩家在各种网络环境下都能获得良好的游戏体验

### 5. API测试与压力测试
- `fiddler_apifox/`：包含Fiddler抓包分析、Apifox API测试和JMeter压力测试
- **测试过程**：先通过Fiddler抓包获取API信息，再使用Apifox进行功能和安全测试，最后用JMeter进行压力测试
- **考量**：验证API的功能正确性、安全性和性能稳定性

### 6. 资源文件
- `Assets/`：存放所有测试过程中产生的截图、录屏和附件
- **包含内容**：Bug截图、测试过程视频、API测试截图、弱网测试录屏等

### 7. 补充文档
- `docs/`：包含SQL查询示例、抓包记录等补充材料

## 测试工具使用
- **Fiddler**：用于API抓包分析
- **Apifox**：用于API功能和安全测试
- **JMeter**：用于性能压力测试
- **Clumsy**：用于模拟弱网环境

## 总结
本项目通过多种测试方法和工具，全面验证了游戏的功能、性能和用户体验。测试过程中注重细节，发现并记录了潜在问题，为游戏质量的提升提供了有力支持。
