# 星穹铁道抽卡模块测试用例

## 1. 测试目标
- 验证抽卡功能的正确性
- 确保抽卡概率符合设计要求
- 测试抽卡结果的展示和奖励发放

## 2. 测试环境
- 游戏版本：v1.0.0
- 操作系统：Windows 10
- 网络环境：稳定网络

## 3. 测试用例

### 3.1 单抽测试
- **测试ID**：TC-001
- **测试名称**：单抽功能测试
- **测试步骤**：
  1. 进入抽卡界面
  2. 点击单抽按钮
  3. 确认消耗物品（星轨专票）
  4. 观察抽卡动画
  5. 查看抽卡结果
- **预期结果**：
  - 成功消耗1张星轨专票
  - 抽卡动画正常播放
  - 抽卡结果正确显示
  - 获得的物品正确发放到背包

### 3.2 十连抽测试
- **测试ID**：TC-002
- **测试名称**：十连抽功能测试
- **测试步骤**：
  1. 进入抽卡界面
  2. 点击十连抽按钮
  3. 确认消耗物品（10张星轨专票）
  4. 观察抽卡动画
  5. 查看抽卡结果
- **预期结果**：
  - 成功消耗10张星轨专票
  - 抽卡动画正常播放
  - 抽卡结果正确显示（至少包含1个4星或以上物品）
  - 获得的物品正确发放到背包

### 3.3 保底机制测试
- **测试ID**：TC-003
- **测试名称**：保底机制验证
- **测试步骤**：
  1. 连续进行9次抽卡（未获得5星物品）
  2. 进行第10次抽卡
  3. 观察抽卡结果
- **预期结果**：
  - 第10次抽卡必定获得5星物品
  - 保底计数正确重置

### 3.4 资源不足测试
- **测试ID**：TC-004
- **测试名称**：资源不足时的处理
- **测试步骤**：
  1. 确保星轨专票数量为0
  2. 尝试进行单抽
  3. 观察系统提示
- **预期结果**：
  - 系统显示资源不足的提示
  - 无法进行抽卡操作

### 3.5 网络异常测试
- **测试ID**：TC-005
- **测试名称**：网络异常时的抽卡处理
- **测试步骤**：
  1. 进入抽卡界面
  2. 开始抽卡操作
  3. 模拟网络中断
  4. 恢复网络连接
  5. 查看抽卡结果
- **预期结果**：
  - 网络恢复后能正确显示抽卡结果
  - 不会重复消耗资源
  - 获得的物品正确发放

### 3.6 抽卡概率验证
- **测试ID**：TC-006
- **测试名称**：抽卡概率符合设计要求
- **测试步骤**：
  1. 进行多次抽卡（建议100次以上）
  2. 统计各稀有度物品的出现次数
  3. 计算实际概率
- **预期结果**：
  - 5星物品概率约为1.6%
  - 4星物品概率约为13%
  - 3星物品概率约为85.4%

### 3.7 新角色UP池测试
- **测试ID**：TC-007
- **测试名称**：新角色UP池概率验证
- **测试步骤**：
  1. 进入新角色UP池
  2. 进行多次抽卡
  3. 统计新角色的出现次数
- **预期结果**：
  - 5星物品中，新角色的概率约为50%
  - 4星物品中，UP角色/光锥的概率约为50%

### 3.8 抽卡记录查看
- **测试ID**：TC-008
- **测试名称**：抽卡记录功能测试
- **测试步骤**：
  1. 进行多次抽卡
  2. 点击抽卡记录按钮
  3. 查看抽卡历史
- **预期结果**：
  - 抽卡记录正确显示
  - 包含抽卡时间、物品名称、稀有度等信息
  - 记录按时间倒序排列
