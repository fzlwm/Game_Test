# 星穹铁道抽卡记录API测试文档

## 测试概述
- **测试目标**：验证抽卡记录API的基本功能和性能
- **测试工具**：Fiddler（抓包）、Apifox（API测试）
- **测试环境**：Windows 10，稳定网络/弱网环境
- **测试时间**：2026-03-19

## API信息
- **API端点**：`https://api.honkai-star-rail.com/game/draw/records`
- **请求方法**：GET
- **认证方式**：Bearer Token
- **主要参数**：player_id, page, page_size, pool_id

## 测试用例

### 1. 正常获取抽卡记录
- **测试ID**：API-DRW-001
- **测试步骤**：
  1. 设置请求参数：player_id=123456, page=1, page_size=20
  2. 发送GET请求
  3. 验证响应
- **预期结果**：
  - 状态码：200
  - 响应包含抽卡记录列表

### 2. 分页测试
- **测试ID**：API-DRW-002
- **测试步骤**：
  1. 设置请求参数：player_id=123456, page=2, page_size=10
  2. 发送GET请求
  3. 验证分页信息
- **预期结果**：
  - 状态码：200
  - 响应包含第2页的记录

### 3. 弱网环境测试
- **测试ID**：API-DRW-003
- **测试步骤**：
  1. 使用Apifox网络模拟，设置延迟300ms，丢包10%
  2. 发送请求
  3. 记录响应时间
- **预期结果**：
  - 响应成功，无超时
  - 响应时间 < 2000ms

### 4. 未授权访问测试
- **测试ID**：API-DRW-004
- **测试步骤**：
  1. 移除Authorization头
  2. 发送GET请求
  3. 验证响应
- **预期结果**：
  - 状态码：401 Unauthorized

### 5. 卡池筛选测试
- **测试ID**：API-DRW-005
- **测试步骤**：
  1. 设置请求参数：player_id=123456, pool_id=101
  2. 发送GET请求
  3. 验证返回记录
- **预期结果**：
  - 状态码：200
  - 所有记录的pool_id为101

### 6. 大数据量测试
- **测试ID**：API-DRW-006
- **测试步骤**：
  1. 设置page_size=100
  2. 发送GET请求
  3. 验证响应时间和数据
- **预期结果**：
  - 状态码：200
  - 响应时间 < 2000ms
